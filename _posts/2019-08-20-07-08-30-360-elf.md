## 课程简介

近日，泰晓科技 与 “Linux 阅码场” 强强联合，首度推出 [『360° 剖析 Linux ELF』](https://www.yomocode.com/courses/9)（简称 “ELF 剖析”） 在线视频课程。

该课程由泰晓科技创始人（前魅族 Linux 内核团队技术总监）吴章金老师主讲。

课程将于 2019 年 10 月 19 日正式在线开课，目前正在开放报名中……

详情请阅读下文进一步了解。

## 课程背景

**关于 “阅码场” 平台**

阅码场平台提供优质的课程，全部都由业界拥有 10 年以上工作经验的一线资深工程师开课，强调内容能直接用于工程实战而不是泛泛而谈。

**关于该 Topic**

ELF 是 Linux 平台上可执行程序文件的标准格式，通过剖析它可以把体系结构、汇编语言、编译原理、链接装载、程序运行、代码库、系统调用、代码优化等基础原理和工程实践很好地串起来。

这些内容在网络上散落一地，但是真正完全串起来，很系统地并且能够以『实验』的方式去讲解的非常少。

**Topic 背景简单回顾**

刚上研那伙，写过一系列博客，后来汇总成《C 语言编程透视》一书并开源。通过这个系列全面了解了进程的来龙去脉，并对程序开发过程的细节、ELF 格式的分析、进程的内存映像等进行了全面地梳理。

[![C 语言编程透视封面](http://tinylab.org/images/courses/cbook-cover.jpg)](https://tinylab.gitbooks.io/cbook)

而且有台湾的黄老师（Jserv）引入到了大学课堂：

![Jserv 老师把《C 编程透视》用到大学课堂](http://tinylab.org/images/courses/cbook-jserv.jpg)

**该 Topic 对工作的重要性**

数年的从业生涯有一个发现：“目前的 Linux 人才看似很多，刚从学校毕业没什么编码经验的学生也可以很快写个小驱动，看似门槛很低，但是真正能够从系统层面解决问题，做了工作不用别人搽屁股的人才很少”。

再回头看高校，很多课程离真正的实践很远，而且相应的实验都是离散的，没有很好地串起来。C 语言编程课上基本只学习了 C 语言的语法，没有了解背后的编译和运行原理。

在这种情况下，解决问题时往往只能看到表象，而且“无知者无畏” —— 可能会对技术的深度和问题的严重性缺少必要的敬畏，一方面解决问题的效率很低，另一方面，解决一个问题挖 N 个坑。

**学完该 Topic 能收获什么**

通过这个 Topic，我们希望能够做到：**一个操作下去，这个背后的运行轨迹要能够像放电影一样，可以随时回放和前进**。

我们不光要能在实际工作中更高效地 Debugging，而且要构建知识体系，并具备未雨绸缪地从源头上消灭问题的能力。

一句话来总结，这个 Topic 就是以 ELF 剖析为切入点，带领大家阅读并实践《程序员的自我修养》和《C 语言编程透视》，从而践行码农的自我修养。

## 讲解主线

**课程讲解主线**

本课程围绕 X86_32 Hello ELF 裁剪，从 Linux 等主流操作系统的可执行文件格式、ELF 规范、X86 体系结构和指令集到 X86 AT&T 汇编语言、代码编译、静态链接、共享库、动态链接、系统调用、程序执行到程序装载、程序运行、内存分布及程序退出。

全程又会根据各个章节需要对各类 ELF 分析、裁剪和编辑工具进行用法详解。

本课程相关章节既相互联系，又保持相对的独立性，可以按需检索。所有章节都有配套实验和作业，可以通过专属微信群探讨，多位一线 Linux 技术专家提供指导。

学习本课程，一方面会在理解程序背后原理的过程中不断享受 “醍醐灌頂的美妙”，另外一方面，切切实实建立起知识体系和图谱，把各个环节做到融汇贯通，在实践中逐步完成程序员的自我修养。

![Linux C 语言编程 OverView](http://tinylab.org/wp-content/uploads/2014/03/c_dev_procedure.jpg)

**课程实验环境**

本课程的实验部分主要基于 [Linux Lab](http://tinylab.org/linux-lab)，Linux Lab 基于 Docker 和 Qemu，当前已支持 7 大处理器架构，15 款虚拟开发板，都提供了预先编译好的交叉编译器、内核和文件系统镜像以及Qemu 启动脚本。

因此无需购买其他硬件，只需一台 PC 或者笔记本，数十分钟内即可搭建。通过 Linux Lab 即可完成相关实验。

本课程以 32 位 X86 处理器为实验对象，由于 Linux Lab 提供了包括 X86, ARM, MIPS, PowerPC, Risc-V, Csky, Loongson 在内的多种处理器系列支持，也为各大处理器提供了 Hello 汇编语言例程，所以大家完全可以在学习完该课程后，尝试在其他处理器平台上完成相关作业，做到触类旁通，举一反三。

[![Linux Lab 实验环境](http://tinylab.org/wp-content/uploads/2016/06/docker-qemu-linux-lab.jpg)](http://showdesk.io/2017-03-11-14-16-15-linux-lab-usage-00-01-02/)

## 整体大纲

**整体目标**：把 C 语言背后各关联知识点通过 ELF 剖析的过程串起来，做到以点带面，融会贯通。

**内容纲要**：

1. 开篇：古有 “庖丁解牛”，今有 “码农剖 ELF”

2. ELF 是什么？
    * Linux 支持哪些可执行文件格式
    * 可执行文件格式的演进历史
    * 各大操作系统都在用什么格式

3. Hello 可执行文件可以有多小？
    * Linux 下默认编译的 Hello.c 有多大？
    * 各大平台上的二进制 Hello 可执行文件做到了多小？
    * 大家都做了哪些尝试？

4. 基础知识筹备
    * ELF 文件格式简介
    * X86 体系结构知识
    * X86 AT&T 汇编语言
    * C 语言函数调用规范
    * ELF 工具套装介绍

5. ELF 程序的生成
    * 代码编译与编译器优化
    * 静态链接和默认链接行为
    * 共享库和动态链接
    * 汇编语言版 Hello

6. ELF 程序的执行
    * 操作系统启动过程简介
    * 按键触发程序执行的那一刻
    * 命令行参数和环境变量
    * 系统调用 Exec

7. ELF 程序的运行与退出
    * ELF 程序加载
    * 进程的内存布局
    * 进程的运行和退出
    * 进程跟踪和调试原理

8. 结语：像一个外科手术专家那样

9. 参考书目一览
